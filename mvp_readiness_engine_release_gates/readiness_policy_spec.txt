Readiness Policy Specification
Version: 2.1

LLM Scope Statement
This policy defines quantitative gate logic for LLM-enabled release candidates.

1. Composite Policy Score
policy_score = (0.30 * groundedness_score) + (0.20 * task_success_rate) + (0.15 * prompt_robustness) + (0.15 * safety_compliance_score) + (0.10 * latency_slo_compliance) + (0.10 * audit_completeness)

2. Required Subscores
groundedness_score = supported_claims / total_claims
task_success_rate = successful_tasks / total_tasks
prompt_robustness = adversarial_tests_passed / adversarial_tests_total
safety_compliance_score = 1 - safety_violation_rate
latency_slo_compliance = compliant_requests / total_requests
audit_completeness = complete_artifacts / required_artifacts

3. Gate Outcomes
Pass: policy_score >= 0.85 and all hard constraints pass.
Conditional Pass: 0.78 <= policy_score < 0.85 with approved exception.
Fail: policy_score < 0.78 or any hard constraint failure.

4. Hard Constraints
- HallucinationRate <= 0.03
- SafetyViolationRate <= 0.01
- Critical vulnerability count = 0
- Required legal/compliance artifacts complete

5. Override Policy
Only designated approvers may override a failed gate. Every override requires:
- explicit business justification
- compensating controls
- expiration date
- signed owner accountability
